# 障害対応（Runbook）

## 1. 影響範囲の把握
- どの画面/機能が影響しているか確認
- 直近のデプロイ有無を確認（Vercel Deployments）

## 2. ログ確認
- Vercel Functions ログで `/api/*` エラーを確認
- クライアント側はブラウザのコンソールログを確認

## 3. よくある原因
- `DATABASE_URL` の設定ミス（接続文字列/環境違い）
- Prisma マイグレーション未適用
- OAuth の `redirect_uri` 不一致

## 4. 復旧手順
- 環境変数の修正 → Redeploy
- DB マイグレーション適用
- 必要に応じてロールバック

## 5. 連絡/共有
- 影響範囲・復旧対応・再発防止を記録する
